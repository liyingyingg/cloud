	虚拟机

3.	容器技术和虚拟化技术的区别？
	虚拟机通常包含整个操作系统及其应用程序，同时也需要运行一个 hypervisor 来控制虚拟机。
因为虚拟机包括操作系统，所有大小一般是几个 G。使用虚拟机的缺点之一是启动操作系统以及初始化托管应用会花费几分钟的时间，但是，容器是轻量级的而且大小在 M 以内。性能方面，与 VM 相比，容器表现更加出色，并且几乎可以秒启动。

4.	容器技术有什么优势？
	容器能与主机的操作系统共享资源
	容器具有可移植性
	容器是轻量的，这意味开发者能同时运行数十个容器，并能模拟分布式系统在真实运行环境下的情况
	对于最终用户以及开发者而言，容器的优势不仅仅体现在云端部署上，用户可以下载并执行复杂的应用程序，而无须花费大量时间在配置和安装上，也无需担心对系统本身的改动。另一方面，应用程序大的开发者不再操心用户环境的差别，及依赖关系是否否满足。

5.	什么是镜像？
	Docker 镜像是一个特殊的文件系统，除了提供容器运行时所需的程序、库、资源、配置等文件外，还包含了一些为运行时准备的一些配置参数（如匿名卷、环境变量、用户等）。镜像不包含任何动态数据，其内容在构建之后也不会被改变。

6.	什么是镜像的分层？
	镜像只是一个虚拟的概念，其实际体现并非由一个文件组成，而是由一组文件系统组成，或者说，由多层文件系统联合组成。镜像构建时，会一层层构建，前一层是后一层的基础。每一层构建完就不会再发生改变，后一层上的任何改变只发生在自己这一层。

27.	什么是数据卷？数据卷的作用是什么？
	数据卷是直接在主机挂载的文件或目录,不属于常规联合文件系统的一部分。
	能够让容器与主机,或容器与其它容器之间轻松共享数据
28.	如何建立一个数据卷？

29.	如何使用其他容器已经创建的数据卷？

30.	什么情况下一个数据卷会被删除？
	• 容器被 docker rm -v 删除
	• docker run 中带有--rm 选项
	• 目前没有容器与该卷关联
	• 该数据卷没有被指定使用主机目录
31.	什么是Docker的守护进程？它的作用是什么？

	Docker 守护进程:负责容器的创建、运行和监控,还负责镜像的构建和存储。守护进程通过 docker daemon 命令启动,一般会交由主机的操作系统负责运行。
